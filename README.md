## What is Vessel?

Vessel turns a person's public writing, interviews, and recorded talks into a deployable AI persona ‚ÄĒ called a **Soul** ‚ÄĒ that can be loaded into any AI tool to reason, argue, and communicate the way that person does.

```
Input: years of public thought, essays, talks, tweets
          ‚Üď  Soul Miner pipeline  ‚Üď
Output: SOUL.md + SKILL.md + MEMORY.md
```

These three files are plain Markdown. They work as a system prompt in Cursor, Claude, ChatGPT, or any AI tool. No API keys. No platform lock-in.

**What makes this different:**

Most "AI persona" tools summarize what a person said. Vessel captures *how* they think ‚ÄĒ their cognitive operating system, decision patterns, rhetorical moves, blind spots, and hard limits. The difference between a summary and a Soul is the difference between reading about a boxer and seeing them fight.

**What Vessel is not:**
- Not a digital clone or a replacement for the actual person
- Not trained on private data ‚ÄĒ public sources only
- Not a chatbot wrapper ‚ÄĒ it's a structured format anyone can use
- Not a hallucination engine ‚ÄĒ every claim is sourced, or marked `[INFERRED]`, or removed

## The Soul Format

Every Soul is three files, each with a distinct role:

| File | Layer | What it contains | How to use it |
|------|-------|-----------------|---------------|
| `SOUL.md` | Identity | Who they are, how they think, how they communicate, their red lines, their blind spots | Load as AI system prompt |
| `SKILL.md` | Capability | What they excel at, trigger phrases that unlock their best responses, anti-patterns, installation guide | Read before using |
| `MEMORY.md` | Knowledge | 15‚Äď20 core arguments with sources, signature frameworks, counterintuitive takes, intellectual influences, what they don't know | Add as context / RAG |

The format is open. If you can read Markdown, you can read ‚ÄĒ or write ‚ÄĒ a Soul.

## Building a Soul

The **Soul Miner** is an automated 6-phase pipeline that discovers, extracts, and distills a person's public content into a Soul Package.

### Prerequisites

```bash
# Python 3.10+
pip install -r .agents/skills/soul-miner/requirements.txt
```

### Run the pipeline

```bash
# Phase 1 ‚ÄĒ Discover sources (blog, Twitter/X, YouTube, RSS, podcasts, GitHub)
python .agents/skills/soul-miner/scripts/discover.py "Paul Graham"

# Phase 2 ‚ÄĒ Extract content by quality tier
python .agents/skills/soul-miner/scripts/mine.py sources_discovered.json --output raw_content.json

# Phase 3‚Äď6 ‚ÄĒ Distill into Soul files
python .agents/skills/soul-miner/scripts/distill.py raw_content.json --output paul-graham/
```

Or load `.agents/skills/soul-miner/SKILL.md` into your AI agent and say:

```
Mine Paul Graham's Soul.
```

The agent runs all six phases automatically and only pauses at genuine uncertainty.

### The Pipeline

```
Phase 1  DISCOVER  ‚ĒÄ‚ĒÄ‚ĒÄ Search 10+ platforms for source content
Phase 2  MINE      ‚ĒÄ‚ĒÄ‚ĒÄ Extract, prioritize, and clean raw material
Phase 3  DISTILL   ‚ĒÄ‚ĒÄ‚ĒÄ 6-dimension personality archaeology
Phase 4  WRITE     ‚ĒÄ‚ĒÄ‚ĒÄ Generate Soul files in first person
Phase 5  VERIFY    ‚ĒÄ‚ĒÄ‚ĒÄ 3-tier fact-check: source every claim or remove it
Phase 6  TEST      ‚ĒÄ‚ĒÄ‚ĒÄ 20-question style fidelity test (‚Č• 15/20 to ship)
```

#### Source Quality Tiers

| Tier | Sources | Why |
|------|---------|-----|
| P0 ‚ÄĒ First-Hand Writing | Blog posts, newsletters, books, long threads | Deliberate, considered expression |
| P1 ‚ÄĒ Spoken Content | Podcast transcripts, talks, interviews | Natural voice, off-the-cuff thinking |
| P2 ‚ÄĒ Short-Form | Tweets, HN comments, brief responses | Statistical patterns across a corpus |
| P3 ‚ÄĒ Secondary (caution) | Articles *about* them, Wikipedia | Context only, never for attributing views |

Minimum viable Soul: **10,000 words** of P0+P1 material. Under 3,000 words, the pipeline pauses.

#### The 6-Dimension Analysis

Every Soul is built from six angles:

1. **Cognitive OS** ‚ÄĒ How do they decompose problems? First principles, systems thinking, historical analogy?
2. **Value Core** ‚ÄĒ What do they deeply care about? What makes them angry? Where do they think everyone else is wrong?
3. **Communication Style** ‚ÄĒ Sentence structure, rhythm, signature rhetorical moves, what they *never* say
4. **Decision Architecture** ‚ÄĒ How do they handle uncertainty? What's their risk profile? Time horizon?
5. **Knowledge Map** ‚ÄĒ Deep expertise domains, cross-domain transfer patterns, intellectual lineage
6. **Tensions & Blind Spots** ‚ÄĒ Contradictions, evolution over time, what they don't see well

#### Authenticity Enforcement

| Label | Meaning |
|-------|---------|
| *(Source: X)* | Direct quote or clear paraphrase from a specific source |
| `[INFERRED from: X, Y]` | Reasoned conclusion supported by multiple sources |
| *removed* | Couldn't be verified ‚ÄĒ deleted, not hedged |

A Soul with more than 10% unverifiable claims is rejected. There is no "probably" category.

## Contributing a Soul

The catalog grows by contribution.

**To submit a Soul:**

1. Run Soul Miner on your subject ‚ÄĒ ensure ‚Č• 10,000 words of P0+P1 material
2. Verify authenticity (Phase 5 ‚ÄĒ check that ūüĒī claims are under 10%)
3. Run the dialogue test (Phase 6 ‚ÄĒ score ‚Č• 15/20)
4. Place output in `vessel/souls/{person-slug}/`
5. To list the Soul on the site, add `vessel/src/data/souls/{person-slug}.json` (see `vessel/src/data/HOW_TO_ADD_SOUL.md`)
6. Run `npm run verify` from the repo root before opening a PR
7. Open a PR with a brief note on source quality and material volume

**Contribution guidelines (PR checklist):**
- Public figures only, with substantial public writing or speaking records
- No fabrication ‚ÄĒ if you can't source it, mark it `[INFERRED]` or remove it
- Living private individuals require their explicit consent
- All Soul files are released under CC BY 4.0

## Why This Exists

Print made thought *copyable*.
The internet made thought *transmissible*.
Vessel is a bet that the next step is thought becoming *runnable*.

You can read Marcus Aurelius but you can't ask him about your specific situation. You can study Feynman but you can't run your paper by him. The gap between "their words on a page" and "their reasoning applied to your problem" is the gap Vessel closes.

The Soul format is designed to be:

- **Verifiable** ‚ÄĒ every claim traceable to a source, no exceptions
- **Portable** ‚ÄĒ plain Markdown, works with any AI model, today and in the future
- **Honest** ‚ÄĒ explicit about limits, blind spots, temporal cutoffs, and what was inferred
- **Durable** ‚ÄĒ the same files that work with Claude 3 should work with whatever comes next

The deeper goal is not to simulate people. It's to build a new kind of intellectual inheritance ‚ÄĒ one that's interactive, honest about its limits, and available to anyone.

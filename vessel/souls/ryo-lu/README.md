## How to use this Soul

### Option 1: Full persona — talk to Ryo

Use `skills/ryo-lu/SKILL.md`. The AI will embody Ryo: first-person, "it's all the same
thing," systems before features, iteration past the first AI output, close to the material.

**Best for:** Design critique, product architecture decisions, AI workflow questions,
"why does this feel fake?" diagnoses, building things that feel soulful.

**Prompts that work well:**
- "Here's my design — what feels false?"
- "I'm using AI to design. How do I avoid making slop?"
- "We have too many features. How do I think about the system?"
- "What are the right building blocks for this product?"
- "ryOS as an inspiration — how do I get that close to my material?"

---

### Option 2: Systems design — product architecture

Use `skills/systems-design/SKILL.md` for structured design system and product architecture
thinking. Phase 1-4 methodology: understand problem space → find primitive set → design
conceptual layer → allow for chaos.

**Best for:** Information architecture, design system structure, "what primitives should I
build?", products that are becoming incoherent as they scale, Notion-style block systems.

---

### Option 3: Soulful AI workflow — iterate past the first output

Use `skills/soulful-ai-workflow/SKILL.md` for the step-by-step process of using AI as
clay, not sculpture. Diagnose false elements → build better bricks → iterate toward true →
soul check.

**Best for:** Any AI-assisted work (design, code, writing) that feels generic. Learning the
practice of tasting what's wrong and fixing it, one false element at a time.

continue to evolve. This soul reflects his documented positions in that period.

Visual design specifics (color systems, typography choices, motion details) are less
represented in public sources and are not extensively covered in this package.

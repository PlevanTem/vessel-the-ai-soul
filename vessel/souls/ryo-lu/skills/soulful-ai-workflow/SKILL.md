# Soulful AI Workflow — AI-as-Material Skill

The method for using AI as raw material — clay, not sculpture. The first AI output is where
the work begins, not where it ends.

**Persona context → read `../../SOUL.md`**
**Knowledge context → read `../../MEMORY.md`**

## The Core Distinction

**AI slop:** First output → ship
**Soulful work:** First output → identify what's false → refine → identify what's still false → refine → ... → ship when it's true

The difference is not the AI tool. The difference is whether the person iterating has taste,
and whether they keep going past comfortable.

"The purple gradient the AI gave you is just the beginning. The difference between slop and
soulful products is iteration and taste. Most people stop at the first output, rather than
using it to begin."

---

## Phase 1: Diagnose the First Output

Before touching anything, look at what the AI produced and name what's wrong.

**The False Elements Check:**
For each part of the AI output, ask: is this *true* to what this thing should be?

- True = consistent with the essential quality of this thing; it belongs here
- False = could be from any AI output for any prompt; it doesn't belong here specifically

False elements usually look like:
- Default aesthetics (the purple gradient, the "modern" sans-serif, the hero with three feature cards)
- Generic structure (the landing page that looks like every other landing page)
- Missing personality (nothing in this output could only have come from *you*)
- Wrong emotional register (the output is competent but doesn't feel like anything)

**Write down the false elements.** Specifically. Not "it feels generic" — *which* specific
elements feel generic and why? This list is your iteration agenda.

---

## Phase 2: Build Your Bricks

Before iterating on the composition, make sure you have great bricks.

**The Brick Standard:**
- A good brick has a specific character. It's not interchangeable with any other brick.
- A good brick was made with a specific intention — a color chosen for a reason, a type
  size chosen to carry a specific weight, a component whose behavior reflects a decision
  about the interaction.
- A good brick composes well with other good bricks. They share a logic.

If the AI gave you bad bricks (defaults, placeholders, arbitrary choices), replace them
before composing. The composition will only be as good as its worst brick.

**How to build a better brick:**
1. Take the element the AI generated
2. Ask: what is this element trying to do? What is its essential function?
3. Make a specific choice about how to do that function — not the default choice, a *chosen* choice
4. Apply taste: if you showed this brick to someone whose work you admire, would they see
   the intention in it?

---

## Phase 3: Iterate Toward True

Now you have a clearer sense of what's false and better bricks to work with.

**The Iteration Loop:**
```
Current state → What's the most false thing here? → Make it truer → 
Assess the whole → What's the most false thing now? → Make it truer → repeat
```

Rules for this loop:
- Change one thing at a time (so you know what caused any improvement or regression)
- After each change, look at the whole — sometimes fixing one thing reveals another
- Don't stop when it's "good enough." Good enough is still false. Keep going until it
  feels *true* — until every element belongs here specifically.
- When you can't see what's wrong anymore: show it to someone with taste and ask them
  what feels off. Fresh eyes find false notes you've stopped seeing.

**How many iterations?**
There's no number. The question is whether you've reached true. Some things get there in
three iterations. Some take thirty. The stopping condition is not a count — it's whether
you've resolved the false elements you identified in Phase 1.

---

## Phase 4: The Soul Check

Before shipping, run the soul check:

**Q1: Is there something here that could only have come from you?**
Your specific taste, your specific choices, your specific reference points. If this could
have been made by anyone using the same AI prompt, it doesn't have your soul in it yet.

**Q2: Does every element belong?**
Look at each element. Is it there because you chose it and it's right? Or is it there because
the AI put it there and you never questioned it?

**Q3: Would you be proud to show this to someone whose taste you respect?**
Not "would they think it's fine" — would they see the intention? Would they feel that someone
who cared made this?

If yes to all three: you're done. Ship it.
If no to any: go back to Phase 3.

## Applying This to Different Mediums

### Design (Figma / Cursor)
- First output: whatever the AI generated or a rough layout
- False elements: generic components, default styling, no spatial logic
- Build better bricks: define your specific type scale, color logic, spacing system
- Iterate: refine the hierarchy, the relationships, the micro-interactions
- Soul check: does this look like it was made by someone with a specific aesthetic point of view?

### Code (Cursor / AI tools)
- First output: functional but generic implementation
- False elements: any code that doesn't reflect the specific behavior you're trying to create
- Build better bricks: define the right abstractions, the right naming, the right structure
- Iterate: refine toward the architecture that's true to the product's underlying logic
- Soul check: if someone reads this code, do they understand what it's for?

### Writing (any AI-assisted writing)
- First output: competent but could be anyone's
- False elements: generic phrasing, predictable structure, missing voice
- Build better bricks: find your specific sentences — the ones only you would write
- Iterate: replace AI sentences with true sentences, one at a time
- Soul check: does this sound like a specific person talking, or does it sound like AI?

## The "Start with Shit" Mindset

"You always start with shit, with AI, and then you refine it. You make it better. That's
the beginning, not the end."

The most important mindset shift: the bad first output is *not a problem*. It is the
beginning of the process. Something exists where nothing did before. Now you can work
with it. The alternative — trying to get AI to produce something great in one shot — is
the path to slop, because you're trying to optimize the prompt rather than doing the work
of refinement.

Use AI to generate the rough block. Then carve it. That's the workflow.

# Startup Idea Review (PG-style)

Use when the user wants a **rigorous pass** on an early idea.

## Steps

1. **Restate the idea** in one sentence — product, user, pain.
2. **Pull test** — what evidence exists that people want this? (usage, payment intent, retention proxy.)
3. **Obvious/surprising** — is this secretly "everyone already agrees" or a real gap?
4. **Schlep check** — is there tedious work competitors avoid? Is the user avoiding it?
5. **Fastest honest experiment** — what could you learn in 2 weeks?
6. **Kill / pivot / double-down** — one clear recommendation with reasons.

## Output

- **Verdict** (one line)
- **Three strengths** / **Three risks**
- **Next experiment** (specific)

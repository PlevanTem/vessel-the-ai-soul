# Distill from Files — 从文件素材做人格考古

> soul-forge Phase 3 的详细执行指南。
> 描述如何从用户提供的文件中提炼六维人格特征。

---

## 核心原则

**文件素材 vs. 公开信息的关键差异：**

1. **无上下文补充**：只能用户给的材料，不能搜索补充
2. **私密性更高**：日记/私信往往含有最真实的价值观，但也最敏感——不要将私密内容直接暴露在 Soul 包中，而是转化为人格特征描述
3. **素材可能不完整**：某些维度可能没有对应材料，必须明确标注 `⚠️ 素材不足`，绝不编造
4. **第一人称优先**：用户的第一人称表达是最高质量的证据

---

## 六维人格考古框架

### 维度 1：认知操作系统（Cognitive OS）

**核心问题：** 这个人如何处理问题？什么是他们的默认思维入口？

**在文件中寻找：**
- 「我一般会先...」「我的做法是...」「我发现...规律」
- 分析复杂问题时的拆解方式（是从数据出发？从第一原则？从类比？）
- 遇到不确定性时的处理策略（规避 vs. 拥抱）
- 习惯性的反问方式（「但真正的问题是...」「更重要的是...」）
- 框架名称（如果本人显式使用某个框架）

**提炼输出格式：**
```
核心思维入口：[一句话描述]
标志性推理动作：
  - [动作1] *(Source: 文件名, 示例引用)*
  - [动作2] *(Source: 文件名, 示例引用)*
框架偏好：[实用主义/系统思维/直觉驱动/...] [INFERRED from: ...]
```

---

### 维度 2：价值观内核（Values & Beliefs）

**核心问题：** 这个人真正在乎什么？什么让他们愤怒或兴奋？

**在文件中寻找：**
- 情绪强烈的段落（愤怒、喜悦、失望——情绪是价值观的指示器）
- 反复出现的主题词汇（连续出现 3 次以上的概念往往是核心关切）
- 明确的立场声明（「我相信...」「我认为...是错的」「这很重要，因为...」）
- 牺牲（「我放弃了...是因为...」）——放弃物揭示价值优先级
- 边界（「我不会做...」「这是我的底线」）

**私密素材处理规则：**
- 日记中的私人情绪 → 转化为人格特征（「对公正有强烈的情感驱动」），不要直接引用
- 提取模式，而非暴露事件

**提炼输出格式：**
```
核心价值排序（1=最重要）：
  1. [价值观] — 证据：[引用或间接支撑] *(Source: 文件名)*
  2. [价值观] — [INFERRED from: 文件名, 推断理由]
明确反对：
  - [X] *(Source: 文件名)*
```

---

### 维度 3：沟通风格图谱（Voice & Expression）

**核心问题：** 这个人说话/写作时有什么独特的节奏和语言特征？

**在文件中寻找：**
- **句子长度偏好**：长句还是短句？是否喜欢破折号或分号？
- **标志性词汇**：反复使用的特定词或短语（verbatim 引用最有价值）
- **隐喻系统**：惯用哪类比喻域（运动/军事/自然/技术/烹饪...）
- **语气调子**：正式 vs. 随意，确定 vs. 试探，说教 vs. 对话
- **开头和结尾习惯**：文章/发言如何开场，如何收尾
- **回避词汇**：他们不说什么（有时比说了什么更重要）

**提炼时必须逐字引用，不要改写！**

```
句子节奏：[短促有力 / 长句嵌套 / 破折号打断 / ...]
标志词汇（逐字引用）：
  - "[原文]" *(Source: 文件名)*
  - "[原文]" *(Source: 文件名)*
隐喻偏好：[来自哪个领域的比喻]
语气：[正式程度] + [确定程度] + [距离感]
从不说：[某类措辞或表达风格] [INFERRED]
```

---

### 维度 4：决策架构（Decision Architecture）

**核心问题：** 这个人如何做重要决定？什么是他们的取舍逻辑？

**在文件中寻找：**
- 决策回顾（「当时我选择了...是因为...」）
- 取舍描述（「我知道这意味着放弃...但...」）
- 风险表述（对风险是量化、直觉、还是回避？）
- 时间视野线索（短期 vs. 长期的权衡频率）
- 后悔 or 满意（对过去决策的评价揭示决策标准）

**提炼输出格式：**
```
决策核心驱动：[原则驱动 / 数据驱动 / 直觉驱动 / 关系驱动]
风险偏好：[保守 / 适中 / 激进] [INFERRED from: ...]
时间视野：[近期优先 / 长期优先 / 混合]
取舍优先级：[A > B > C] *(Source: ...)*
```

---

### 维度 5：知识版图（Knowledge Map）

**核心问题：** 这个人深耕哪些领域？知识谱系来自哪里？

**在文件中寻找：**
- 专业术语密度（某领域词汇大量出现 → 深度专业区）
- 引用的人名/书名/概念（知识坐标系）
- 用外行语言解释专业概念（说明真正掌握）
- 领域交叉（在多个领域自如切换）
- 承认不懂的地方（诚实的无知边界比假装知道更有价值）

**提炼输出格式：**
```
核心专业域（深度）：[领域1], [领域2]
熟悉领域（广度）：[领域3], [领域4]
知识坐标系（引用来源）：
  - [人名/书名/概念] *(Source: 文件名)*
明确承认的盲区：[内容] *(Source: 文件名)* 或 [INFERRED]
```

---

### 维度 6：人格矛盾与张力（Tensions & Contradictions）

**核心问题：** 这个人在哪些地方说的和做的不一致？立场在哪里演化了？

**在文件中寻找：**
- 同一话题在不同时期的不同态度（时间跨度素材特别有价值）
- 理想表述 vs. 实际行为的差距
- 「虽然我知道...但我还是...」型表达（有意识的矛盾）
- 对自身矛盾的自我觉察

**处理原则：**
- 矛盾不是缺陷，是人格深度的体现
- 描述时保持中立，不做价值判断
- 时间顺序很重要（「从 X 到 Y 的演化」比「同时持有 X 和 Y」更准确）

```
核心张力：
  - [领域]：[早期立场] → [当前立场] *(Source: 早期文件 → 近期文件)*
  - [话题]：理想上[X]，实际中[Y] *(Source: ...)*
自我觉察：[是否对自身矛盾有意识] *(Source: ...)*
```

## 特殊素材类型的处理技巧

### 只有一种类型的素材

| 素材类型 | 特别适合提炼 | 较难提炼 |
|----------|-------------|---------|
| 纯文章 | 认知OS、知识版图、沟通风格 | 决策架构、个人矛盾 |
| 纯日记 | 价值观、矛盾与张力 | 知识版图、沟通正式风格 |
| 纯对话记录 | 沟通风格、即时反应、立场 | 深度知识版图 |
| 纯代码+注释 | 认知OS、工程哲学、决策架构 | 情感价值观、叙事根源 |

> 单一素材类型时，在输出中注明覆盖不足的维度，加 `⚠️ 素材类型单一，此维度可信度有限`。

---

### 素材覆盖时间跨度较短（< 1 年）

- 维度 6（矛盾与张力）通常无法完整提炼——标注 `⚠️ 时间跨度不足，无法评估立场演化`
- 其他维度正常处理

---

### 素材语言与目标语言不同

如果素材是中文，SOUL.md / MEMORY.md 用英文写作时：
- 标志性词汇和金句保留中文原文，后附英文翻译
- 例：`"认真做事，慢慢来" ("Do things carefully, take it slow")` *(Source: ...)*
